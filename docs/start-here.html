Start here

Updated Oct 6th 2017, for Truffle v3.4.11 and Solidity v0.4.15.

This is a step-by-step guide for software developers who want to start writing decentralized applications for the Ethereum Virtual Machine. It contains the best, most up-to-date material on the web for reasoning about and writing secure smart contract systems.

Smart Contract Development 101

We know you're eager to begin, but really do take some time to master the fundamentals. Start at the cryptocurrency basics until you can comfortably answer:

  • what public and private keys are
  • why a blockchain needs miners
  • how decentralized consensus is reached

Then move on to the quick Ethereum basics section. After these two primers, you can get hands-on with the code and deploy your first smart contract in under an hour.

Be sure to come back periodically to the smart contract security reference, because smart contract security is hard.

  1. Cryptocurrency basics links to top resources on public key cryptography and blockchains.
  2. Ethereum basics explains the Ethereum Virtual Machine, gas, transactions, and other concepts related to this blockchain.
  3. Setting up your test environment introduces testrpc and Truffle.
  4. Writing your first contract gets you to code, interact with, and deploy Proof Of Existence, a neat application of smart contracts.
  5. Writing a secure contract teaches you how vulnerabilities can easily creep in your code.
  6. Using web3 walks you through to interacting with Ethereum and smart contracts from a web dapp.

Smart Contract Security Reference

At OpenZeppelin, we are committed to the growth and prosperity of the decentralized ecosystem, and thus want every smart contract that handles real money to be secure. If you're a new developer to the space, we ask that you read, internalize, and widely share this reference to raise the safety bar of the industry.

  • Principles introduces general, high-level mandates when developing smart contracts.
  • Code patterns gives specific recommendations on how to structure your Solidity code.
  • System architecture provides guidance on organizing multiple contracts that interact with each other.
  • Types of vulnerabilities categorizes frequently-exploited code issues into different types.
